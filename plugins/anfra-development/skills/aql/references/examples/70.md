## Bad AQL
```aql
metric total_orders = orders | count(orders.id);
metric top_10_products_last_quarter =
  top(
    10,
    products.id,
    by: total_orders | where(orders.created_at matches @(last quarter)),
    logic: 'dense',
  );
explore {
  dimensions {
    product_name: products.name,
  }
  measures {
    order_count_last_quarter: total_orders,
    order_count_last_year: total_orders | relative_period(orders.created_at, interval(-1 year)),
    order_count_2_years_ago: total_orders | relative_period(orders.created_at, interval(-2 year)),
    order_count_3_years_ago: total_orders | relative_period(orders.created_at, interval(-3 year)),
  }
  filters {
    products.id in top_10_products_last_quarter,
    orders.created_at matches @(last quarter),
  }
  sorts {
    order_count_last_quarter DESC,
    product_id ASC,
  }
}
```

## Why it's bad
* Top 10 is ranked by products.id, but the explore is broken down by products.name, this discrepancy in granularity will yield wrong results.
